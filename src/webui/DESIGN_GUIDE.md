# WebUI 设计规范

## 定位

- `src/webui` 是单页控制台，不是通用管理后台。
- 这套 WebUI 以用户提供的原始控制台 HTML 为源头，Django 版只做结构迁移、真实数据接入和交互补全。
- 后续修改默认以“保持当前页面风格和信息架构”为优先原则。

## 导航与页面

- 导航固定为 8 页：`总览`、`运行状态`、`连接设置`、`模型设置`、`助手设置`、`表情管理`、`记忆管理`、`你的回忆`。
- 对应 section id 固定为：`page-home`、`page-status`、`page-network`、`page-model`、`page-reply`、`page-emoji`、`page-memory`、`page-recall`。
- 默认不新增新导航层级，不把单页切换改成多路由站点。

## 布局规范

- 整体结构固定为“左侧边栏 + 右侧主内容区”。
- 左侧边栏默认宽度 `260px`，折叠后 `80px`。
- 左侧边栏外轮廓不做大圆角，保持利落直边。
- 右侧主区由顶部标题栏和下方滚动工作区组成。
- 顶部标题栏需与左侧 `WebUI` 顶栏在视觉上对齐。
- 工作区使用 12 列网格，内容最大宽度 `1400px`。

## 视觉风格

- 整体风格是浅色云母质感控制台，不改成通用后台模板。
- 背景只保留左上角柔和光晕，不叠加复杂花纹。
- 卡片统一使用半透明白色云母材质：轻模糊、轻边框、轻阴影、大圆角。
- 字体统一使用 `"Segoe UI Variable", "Segoe UI", sans-serif`。
- 图标统一使用 Font Awesome。

## 颜色规范

- 默认主色是 `rgb(255, 77, 143)`。
- 当前只允许 4 组全局主题色：`255, 77, 143`、`77, 148, 255`、`16, 185, 129`、`139, 92, 246`。
- 激活导航统一使用“主色文字 + 主色浅底 + 左侧细色条”。

## 首页规范

- 首页是 `总览`，只放当前状态和最必要的指标，不做长表单。
- 首页结构固定为：`助手卡`、`状态摘要卡`、`三张统计卡`。
- `助手卡` 显示头像、助手名称、最近同步时间。
- 头像可点击上传，流程固定为：选图 -> 1:1 裁切 -> 上传。
- `状态摘要卡` 只放 `运行状态`、`连接状态`、`运行时长`。
- 三张统计卡固定为：`收到消息`、`发出回复`、`异常次数`。
- `助手未连接` 提示卡只在首页出现。

## 页面内容规范

### 运行状态
- 主要用来看助手是否在正常工作。
- `运行状态 / 连接状态 / 运行时长` 这类状态卡的主文字要比纯数字统计卡小一档。

### 连接设置
- 只放机器人连到 QQ 所需的必要地址项。

### 模型设置
- 使用网格卡片布局，桌面端每行两张。
- 每张模型卡只保留最必要的字段：地址、模型名、密钥。
- 密钥显示规则固定为：开头 3 位 + 中间掩码 + 结尾 5 位。
- 密钥输入框右侧内嵌保存状态图标，不放到卡片头部。
- 密钥占位文案固定为：`留空为不启用`。
- 未配置状态统一显示为：`未启用`。
- 识图模型不再提供手动勾选启用开关，是否启用由配置完整性决定。

### 助手设置
- 用来定义助手名字、说话风格和群聊策略。
- `群聊策略` 使用点击弹出的卡片选择器，不用原生下拉框。
- 选项后面的 `?` 图标用来显示帮助说明。

### 表情管理
- 页面结构固定为：上方一条横向设置区 + 下方一张大卡片展示表情库。
- 不做零散小卡片拼贴布局。

### 记忆管理
- 顶部保留必要的记忆设置项。
- `读取范围` 使用和 `群聊策略` 一样的卡片选择器。
- 下半部是一张大卡片，内部再按网格排布记忆卡片。
- 记忆按分类展示：`重要记忆`、`群聊记忆`、`私聊记忆`。
- 每张记忆卡片支持编辑和删除，同时显示最基本的归属信息。

### 你的回忆
- 当前是独立页面，不再使用右侧抽屉。
- 现阶段仍是占位页，用来预留未来的回忆时间线。

## 按钮与控件

- 设置页的 `保存设置` 按钮统一靠右，宽度收紧，不做得过大。
- 普通操作优先使用当前 `btn-outline` 风格。
- 勾选类控件统一使用“圆形底 + Font Awesome 字体图标”的表现。
- 未选中是圆形浅底 + `minus`，已选中是主色实心圆底 + 白色 `check`。
- 密钥保存状态图标和其他勾选控件必须使用同一套图标语言。

## 交互规范

- 前端继续使用原生 JavaScript，不引入 React、Vue 等状态层。
- 必须保留的基础交互包括：边栏折叠、亮暗切换、主题色切换、section 切换、重启确认弹窗、头像裁切上传、卡片式选择器。
- 帮助提示统一使用自定义悬浮提示，不依赖浏览器默认 title 气泡作为主体验。
- 本地界面偏好继续保存在 `localStorage`。

## 文案规范

- 文案要短、通俗、好懂。
- 优先用“助手”“怎么连”“怎么说话”“会不会记住”这类自然说法。
- 不默认使用“运行态”“配置校验”“接口持久化”等偏内部的术语。
- 保存成功、失败、重启成功、上传成功等反馈都要像对用户说话。
- 密钥为空时的统一说法是：`留空为不启用`。

## 真实功能规范

- `总览` 和 `运行状态` 以运行快照为主数据源。
- 设置页读写真实 `config.json`。
- 头像使用真实上传接口和真实文件存储。
- `重启服务` 是真实功能，含义固定为：只重启 bot 后端，不重启 Django WebUI，页面保持在线。

## 不要做的事

- 不要把当前控制台改成通用后台模板。
- 不要随意增加新的导航层级。
- 不要把单页切换改成真正的多页跳转。
- 不要引入新的前端框架。
- 不要把首页做成系统资源监控大盘。
- 不要为了“更现代”就随意改卡片顺序、间距、密度或信息层级。

## 改动流程

1. 先看这次改动是不是还在这份规范里。
2. 能只改数据和文案，就不要先改页面骨架。
3. 能复用现有卡片、按钮、提示样式，就不要另起一套组件语言。
4. 如果确实要突破这份规范，先更新文档，再改页面。
